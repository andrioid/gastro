# Decisions

- **2026-03-31** (m+git@andri.dk) SSE support implemented as a generic `pkg/gastro/sse.go` helper (framework-agnostic) with a separate `pkg/gastro/datastar/` subpackage for Datastar-specific sugar. No external dependencies added -- SSE protocol is ~90 lines. No compiler/codegen changes needed; SSE endpoints are plain Go handlers registered alongside gastro routes.
- **2026-03-31** (m+git@andri.dk) Component composition enabled -- components can now `use` other components. Fixed `componentTmpl` in `generate.go` to wire up FuncMap for `use` declarations, mirroring the existing `handlerTmpl` pattern. Parser and template transformer already supported this; only code generation was missing.
- **2026-03-31** (m+git@andri.dk) `gastro.Render` struct pattern for type-safe component rendering. Generated `render.go` provides `Render.ComponentName(props)` methods that wrap internal component functions. Props type aliases exported per component. Render methods call internal functions (preserving frontmatter logic) rather than bypassing them. Components without Props get zero-argument methods. Components with slots get variadic `children` parameter.
- **2026-03-31** (m+git@andri.dk) LSP: component prop validation, signature hover, go-to-definition. Cross-file component resolution reads Props struct from component .gastro files (cached, invalidated on change). Missing props are warnings (severity 2) since Go zero-values may be valid; unknown props are errors. Go-to-definition remaps all three LSP response formats (Location, Location[], LocationLink[]). Component hover shows the full Props struct.
- **2026-03-31** (m+git@andri.dk) Embedding (Phase 7): Templates are now loaded from `fs.FS` at runtime rather than inlined as string literals. `Routes()` parses all templates into a registry using a centralized `__gastro_buildFuncMap` (with per-template component wiring). In dev mode (`GASTRO_DEV=1`) templates are re-parsed from disk on every request for live reload without restart. `WithFuncs` is now fully wired. Static assets are embedded via `//go:embed`; static/ is copied (not symlinked) into `.gastro/static/` because Go's embed does not follow directory symlinks. `embed.go` is generated alongside other `.gastro/` output.
- **2026-04-01** (m+git@andri.dk) Concurrency hardening: fixed `devReloader.Stop()` double-close race (replaced select/default with `sync.Once`), added `closed` guard to `GoplsProxy.writeMessage` so `Close()` is safe during in-flight requests, added concurrent stress tests for Subscribe/Broadcast/Stop and SSE.Send, created `mise run test` task that enforces `-race` flag.
- **2026-04-01** (m+git@andri.dk) Compression: documented as user-space concern rather than built-in. Gastro stays zero-dependency; users wrap `Routes()` with their own middleware or use a reverse proxy. Added `docs/compression.md` with stdlib gzip middleware example and caveats (skip SSE, skip already-compressed formats).
- **2026-04-01** (m+git@andri.dk) Component syntax modernization: replaced `use Layout "path"` with `import Layout "path.gastro"` (component imports distinguished by `.gastro` extension, coexist in standard `import (...)` blocks). Replaced `gastro.Props[T]()` with `gastro.Props()` (generic type parameter removed; `type Props struct` is sufficient). Both forms supported: `Title := gastro.Props().Title` for direct access, `p := gastro.Props()` for whole-struct assignment. Frontmatter rewriter refactored from string-based to AST-based detection (eliminates false positives in strings/comments). Added validation: rejects missing Props struct, multiple Props declarations, non-struct Props, mixing Context+Props. Legacy syntax (`gastro.Props[T]()`, `use` keyword) removed.
- **2026-04-01** (m+git@andri.dk) LSP: component auto-import and prop completions. Added `<` as completion trigger character. Components directory is scanned on startup to build an index of available components. Selecting an un-imported component from completions auto-inserts the import declaration via `additionalTextEdits`. Prop name completions offered when cursor is inside a component tag (`<Card |/>`), with type info and filtering of already-specified props.
- **2026-04-01** (m+git@andri.dk) LSP: pipe function completions in prop values. Added `|` as trigger character. When cursor is after `|` inside a prop value expression (`<Card Date={.X | }`), only template functions are suggested (not variables or component names). Detection uses composable helper functions (`isInsideComponentTag`, `isInsideUnclosedBrace`, `isAfterPipeOutsideQuotes`) instead of a complex regex. Variable completions inside `{.}` already worked via fall-through from the existing `.` trigger.
- **2026-04-02** (m+git@andri.dk) Aligned Go versions across the project: `go.mod` (root + 3 examples) bumped from `go 1.26.0` to `go 1.26.1`, `mise.toml` pinned to `1.26.1` (was floating `1.26`). `go mod tidy` removed redundant `toolchain go1.26.1` directives since they match the `go` directive exactly. Fixes `compile: version "go1.26.0" does not match go tool version "go1.26.1"` errors from stale build cache.
- **2026-04-02** (m+git@andri.dk) Component syntax migration: replaced HTML-like `<Component />` / `<Component>...</Component>` / `<slot />` with Go template actions `{{ render Component (dict ...) }}` / `{{ wrap Component (dict ...) }}...{{ end }}` / `{{ .Children }}`. Eliminates ambiguity between component invocations and HTML content. Template transformer rewritten from regex-on-HTML (~250 lines) to `{{ }}` action matching (~180 lines) with a state-aware scanner for correct `{{ end }}` matching. Escape directive removed (no longer needed). Breaking change: all `.gastro` files migrated. LSP updated to detect new syntax.
- **2026-04-02** (m+git@andri.dk) Fix: unique child template names in `replaceWithChildren`. Using the same component-with-children multiple times on one page (e.g., multiple `<Layout>...</Layout>`) produced duplicate `{{define "layout_children"}}` blocks, rejected by Go's template parser. Fix adds an incrementing counter to produce `layout_children_0`, `layout_children_1`, etc. Counter is scoped to `replaceComponents` and passed as `*int` to `replaceWithChildren`.
- **2026-04-02** (m+git@andri.dk) Fix: backtick-aware import extraction in parser. `extractImports` and `stripImports` did line-based matching without tracking whether a line was inside a backtick raw string literal. Import statements inside Go string constants were falsely extracted as real imports, and `stripImports` silently corrupted the string content. Fix adds `inString` tracking using the existing `hasUnclosedBacktick` helper, matching the pattern already used in `splitSections`.
- **2026-04-02** (m+git@andri.dk) Template escape directive: added `{{/* gastro:escape */}}...{{/* /gastro:escape */}}` to skip component/slot transformation inside marked blocks. Uses Go template comment syntax so it's invisible at render time. Null-byte-delimited placeholders prevent collision with user content. Validates unclosed, nested, and orphaned blocks. Primary use case: documentation and code examples that show Gastro component syntax without it being processed by the transformer.
- **2026-04-02** (m+git@andri.dk) Codegen template unification: evaluated and rejected. Keeping `handlerTmpl` (pages) and `componentTmpl` (components) as separate Go templates in `generate.go`. Pages stream directly to `http.ResponseWriter`; components buffer into `template.HTML`. Merging would force pages to buffer unnecessarily and add conditional complexity for ~10 lines of shared code. Fixed a real bug instead: both templates now handle `template.Execute` errors that were previously silently discarded.
- **2026-04-02** (m+git@andri.dk) Bare function calls for components: replaced `{{ render Component (dict ...) }}` with `{{ Component (dict ...) }}`. Leaf components are now standard Go template function calls -- no transformation needed. `wrap` keyword stays for components with children. FuncMap registration uses PascalCase names directly (`fm["Card"]`) instead of `__gastro_` prefix. Added enhanced error messages for undefined PascalCase functions ("did you forget to import it?") and collision warnings when user `WithFuncs()` names shadow component names. LSP updated with AST-based component detection.
- **2026-04-02** (m+git@andri.dk) LSP: AST-based component prop validation. Refactored `DiagnoseComponentProps`, `DetectComponentTagContext`, and `DetectPropValueContext` from regex-only to AST-primary with regex fallback. Fixes: bare component calls (`{{ Card }}`) now emit missing-prop warnings; nested parentheses in dict args (`(dict "Date" (.X | timeFormat "Jan"))`) no longer break prop extraction. Regex fallbacks also improved to handle bare calls (for completions during editing when the template is syntactically incomplete). Design guideline added: prefer AST over regex for template analysis in the LSP.
- **2026-04-03** (m+git@andri.dk) Release automation: adopted conventional commits + release-please for trunk-based development with automatic versioning. SemVer with `bump-minor-pre-major` (breaking changes only bump minor while on v0.x). release-please creates a Release PR on each push to main; merging it tags and creates a GitHub Release. Old `release.yml` (manual `v*` tag trigger + `softprops/action-gh-release`) replaced by `release-please.yml`. Removed `dtolnay/rust-toolchain` (Ubuntu runners include rustup) and `softprops/action-gh-release` (replaced by `gh release upload`). Only approved third-party actions: `jdx/mise-action` and `googleapis/release-please-action`.
- **2026-04-03** (m+git@andri.dk) VS Code extension distribution: platform-specific `.vsix` packages bundling the correct `gastro-lsp` binary per OS/arch. Published to both VS Code Marketplace (`VSCE_PAT`) and Open VSX (`OVSX_PAT`). Version kept in sync with the Go module via release-please `extra-files`.
- **2026-04-03** (m+git@andri.dk) Version embedding: both `gastro` and `gastro-lsp` binaries accept `--version` / `version` / `-v` flags. Version injected at build time via `-ldflags "-X main.version=..."` using `GASTRO_VERSION` env var (defaults to `dev`).
- **2026-04-03** (m+git@andri.dk) Compile-time `{{ raw }}...{{ endraw }}` blocks. Content inside is escaped for literal display: template delimiters (`{{`/`}}`) and HTML characters (`<`, `>`, `&`). Whitespace around markers is always trimmed (no opt-out). Runs in `TransformTemplate` after comment extraction, before wrap transformation. No runtime function registered. LSP stubs prevent parse errors; `stripRawBlocks` prevents false diagnostics. Code examples in the example site inlined directly into `.gastro` files, replacing the `content/docs.go` + `CodeBlock` component pattern.
- **2026-04-04** (m+git@andri.dk) Consolidated `gastro-lsp` into `gastro lsp` subcommand (single binary). Release archives are now `.tar.gz`/`.zip` per platform with SHA256SUMS. VS Code extension is universal (no bundled binary, finds `gastro` on PATH). Zed extension downloads archives. Scaffolded `go.mod` uses the version of the `gastro` binary that created it.
- **2026-04-05** (m+git@andri.dk) LSP: fix deadlock in `invalidateComponentPropsCache` (re-entrant `dataMu` lock), graceful degradation when gopls is missing (instance kept alive with `gopls=nil`), and VS Code gopls install prompt. Instance initialization moved outside `dataMu.Lock()` to avoid blocking during I/O. Custom `gastro/goplsNotAvailable` notification sent once to the editor; VS Code extension intercepts it and offers one-click `go install golang.org/x/tools/gopls@latest` via terminal.
- **2026-04-05** (m+git@andri.dk) VS Code extension: added esbuild bundler to inline `vscode-languageclient` into `dist/extension.js`. Fixes published `.vsix` missing `node_modules/` (`.vscodeignore` excluded them but no bundler existed). CI updated to run `npm run build` before `vsce package`.
- **2026-04-05** (m+git@andri.dk) VS Code extension: migrated from plain JS to strict TypeScript (`src/extension.ts`). Modernized from deprecated `client.onReady()` to v9 async pattern (register notifications before `await client.start()`). Added `npm run typecheck` script. esbuild compiles TS; `tsc --noEmit` is type-check only.
