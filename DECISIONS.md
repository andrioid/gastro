# Decisions

- **2026-03-31** (m+git@andri.dk) SSE support implemented as a generic `pkg/gastro/sse.go` helper (framework-agnostic) with a separate `pkg/gastro/datastar/` subpackage for Datastar-specific sugar. No external dependencies added -- SSE protocol is ~90 lines. No compiler/codegen changes needed; SSE endpoints are plain Go handlers registered alongside gastro routes.
- **2026-03-31** (m+git@andri.dk) Component composition enabled -- components can now `use` other components. Fixed `componentTmpl` in `generate.go` to wire up FuncMap for `use` declarations, mirroring the existing `handlerTmpl` pattern. Parser and template transformer already supported this; only code generation was missing.
- **2026-03-31** (m+git@andri.dk) `gastro.Render` struct pattern for type-safe component rendering. Generated `render.go` provides `Render.ComponentName(props)` methods that wrap internal component functions. Props type aliases exported per component. Render methods call internal functions (preserving frontmatter logic) rather than bypassing them. Components without Props get zero-argument methods. Components with slots get variadic `children` parameter.
- **2026-03-31** (m+git@andri.dk) LSP: component prop validation, signature hover, go-to-definition. Cross-file component resolution reads Props struct from component .gastro files (cached, invalidated on change). Missing props are warnings (severity 2) since Go zero-values may be valid; unknown props are errors. Go-to-definition remaps all three LSP response formats (Location, Location[], LocationLink[]). Component hover shows the full Props struct.
- **2026-03-31** (m+git@andri.dk) Embedding (Phase 7): Templates are now loaded from `fs.FS` at runtime rather than inlined as string literals. `Routes()` parses all templates into a registry using a centralized `__gastro_buildFuncMap` (with per-template component wiring). In dev mode (`GASTRO_DEV=1`) templates are re-parsed from disk on every request for live reload without restart. `WithFuncs` is now fully wired. Static assets are embedded via `//go:embed`; static/ is copied (not symlinked) into `.gastro/static/` because Go's embed does not follow directory symlinks. `embed.go` is generated alongside other `.gastro/` output.
- **2026-04-01** (m+git@andri.dk) Concurrency hardening: fixed `devReloader.Stop()` double-close race (replaced select/default with `sync.Once`), added `closed` guard to `GoplsProxy.writeMessage` so `Close()` is safe during in-flight requests, added concurrent stress tests for Subscribe/Broadcast/Stop and SSE.Send, created `mise run test` task that enforces `-race` flag.
- **2026-04-01** (m+git@andri.dk) Compression: documented as user-space concern rather than built-in. Gastro stays zero-dependency; users wrap `Routes()` with their own middleware or use a reverse proxy. Added `docs/compression.md` with stdlib gzip middleware example and caveats (skip SSE, skip already-compressed formats).
- **2026-04-01** (m+git@andri.dk) Component syntax modernization: replaced `use Layout "path"` with `import Layout "path.gastro"` (component imports distinguished by `.gastro` extension, coexist in standard `import (...)` blocks). Replaced `gastro.Props[T]()` with `gastro.Props()` (generic type parameter removed; `type Props struct` is sufficient). Both forms supported: `Title := gastro.Props().Title` for direct access, `p := gastro.Props()` for whole-struct assignment. Frontmatter rewriter refactored from string-based to AST-based detection (eliminates false positives in strings/comments). Added validation: rejects missing Props struct, multiple Props declarations, non-struct Props, mixing Context+Props. Legacy syntax still supported for backward compat.
- **2026-04-01** (m+git@andri.dk) LSP: component auto-import and prop completions. Added `<` as completion trigger character. Components directory is scanned on startup to build an index of available components. Selecting an un-imported component from completions auto-inserts the import declaration via `additionalTextEdits`. Prop name completions offered when cursor is inside a component tag (`<Card |/>`), with type info and filtering of already-specified props.
- **2026-04-01** (m+git@andri.dk) LSP: pipe function completions in prop values. Added `|` as trigger character. When cursor is after `|` inside a prop value expression (`<Card Date={.X | }`), only template functions are suggested (not variables or component names). Detection uses composable helper functions (`isInsideComponentTag`, `isInsideUnclosedBrace`, `isAfterPipeOutsideQuotes`) instead of a complex regex. Variable completions inside `{.}` already worked via fall-through from the existing `.` trigger.
- **2026-04-02** (m+git@andri.dk) Aligned Go versions across the project: `go.mod` (root + 3 examples) bumped from `go 1.26.0` to `go 1.26.1`, `mise.toml` pinned to `1.26.1` (was floating `1.26`). `go mod tidy` removed redundant `toolchain go1.26.1` directives since they match the `go` directive exactly. Fixes `compile: version "go1.26.0" does not match go tool version "go1.26.1"` errors from stale build cache.
